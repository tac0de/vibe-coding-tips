# UI Vibe Coding

UI 작업에서 중요한 건 “예쁘게”가 아니라 “어떤 장면을 만들지”를 잠그는 것이다.

중요 원칙:

- 첫 화면의 주인공은 하나만 둔다
- 카드, KPI, pill, dashboard 문법이 주인공이 되면 실패다
- 모바일 기준으로 먼저 설계한다
- reference와 함께 금지할 UI 문법을 꼭 준다

Tailwind에서 특히 볼 것:

- spacing hierarchy
- typography hierarchy
- component depth
- surface / border / shadow 규칙
