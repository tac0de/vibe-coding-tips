언제 쓰는지

기존 프로젝트의 디자인 시스템 흔적을 먼저 읽고 그 위에서 작업하고 싶다.

기대 출력

  • 현재 프로젝트의 시각 규칙이 정리된다
  • spacing, type, depth가 함께 잡힌다
  • 새 UI를 기존 톤에 맞출 기준이 생긴다

주의 포인트

  • tailwind class만 나열한다
  • 토큰처럼 보이는 값을 억지로 일반화한다
  • 현재 구조보다 이상적인 시스템만 말한다
