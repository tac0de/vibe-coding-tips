# Refactor Information Hierarchy

## 상황

기능은 유지하고 정보 구조만 정리하고 싶다.

## 바로 복붙할 프롬프트

```txt
현재 UI가 복잡하다.
기능은 유지하고 정보 구조만 정리해라.

목표:
- 우선순위 높은 정보만 남기기
- 타이포 위계 재정리
- 컴포넌트 depth 줄이기
- 모바일 첫 화면 가독성 확보

새 컴포넌트 남발 금지.
먼저 어떤 요소를 제거/축소/유지할지 말한 뒤 수정해라.
```

## 좋은 출력의 기준

- 제거가 먼저다

## 실패 패턴

- 구조가 더 복잡해진다
