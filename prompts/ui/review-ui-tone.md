# Review UI Tone

## 상황

코드는 맞는데 미감이 구린지 확인하고 싶다.

## 바로 복붙할 프롬프트

```txt
이 UI를 리뷰해라.
칭찬보다 아래를 먼저 말해라.

1. 왜 제품 대시보드처럼 보이는지
2. 왜 장면이 약한지
3. 모바일에서 가장 먼저 망하는 부분
4. 지금 당장 제거해야 할 UI 요소 3개
5. 다음 패치가 어떤 순서여야 하는지
```

## 좋은 출력의 기준

- 구린 이유를 구조적으로 말한다

## 실패 패턴

- 추상적인 감상만 말한다

## 다음 액션

제거 패치부터 한다.
