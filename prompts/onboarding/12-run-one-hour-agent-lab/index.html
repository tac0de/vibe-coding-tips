언제 쓰는지

1시간짜리 실습 세션을 실제 프로젝트 기준으로 운영할 때, 서브에이전트와 문서 흐름을 한 번에 묶는 프롬프트.

기대 출력

  • 1시간 안에 끝나는 현실적인 세션이다
  • 역할 분리가 뚜렷하다
  • UI와 D3가 둘 다 “작은 적용” 수준으로 들어간다

주의 포인트

  • 이론 설명만 길고 실제 적용이 없다
  • builder 혼자 다 하게 만든다
  • D3 범위가 커져서 세션이 무너진다

실행 체크

진행 중

게이미피케이션이 아니라 워크플로 마커로 완료 체크를 사용한다.
