# 02. Lock Scope

## 상황

첫 분석은 끝났고, 이제 작업 범위를 좁혀야 한다.

## 바로 복붙할 프롬프트

```txt
좋다. 이제 이번 작업의 수정 범위를 고정한다.

수정 가능:
- [여기에 경로]

수정 금지:
- [여기에 경로]

이번 패치 목표:
- [목표 1개]

먼저 어떤 파일을 어떻게 바꿀지 5줄 안으로만 말해라.
아직 구현하지 말라.
```

## 좋은 출력의 기준

- 목표가 1개다
- 수정 가능/금지 범위가 명확하다

## 실패 패턴

- 범위를 넓게 잡는다
- 목표를 여러 개 넣는다
