# PROMPT_SPEC.md

## 목적

이 문서는 `prompts/` 아래 모든 파일이 같은 형식과 밀도를 유지하게 만드는 단일 스펙이다.

## 파일 형식

모든 프롬프트 파일은 아래 순서를 유지한다.

1. 제목
2. `## 상황`
3. `## 바로 복붙할 프롬프트`
4. `## 좋은 출력의 기준`
5. `## 실패 패턴`
6. `## 다음 액션`

## 제목 규칙

- 역할 프롬프트는 `Builder`, `Explorer`처럼 역할명 그대로 쓴다
- 순서형 프롬프트는 `01. ...`, `02. ...` 형식을 유지한다
- UI/D3 task형 프롬프트는 짧은 영어 제목을 쓴다

## 프롬프트 블록 규칙

- 항상 fenced code block 안에 넣는다
- 그대로 복붙 가능한 완성 입력문으로 쓴다
- 가능하면 아래 4개를 포함한다
  - 목표
  - 제약
  - 금지사항
  - 출력 형식
- 너무 많은 선택지를 한 번에 요구하지 않는다

## 길이 규칙

- `상황`: 1~2문장
- `바로 복붙할 프롬프트`: 8~25줄 권장
- `좋은 출력의 기준`: 2~5개 bullet
- `실패 패턴`: 2~5개 bullet
- `다음 액션`: 다음에 열 파일 또는 다음 프롬프트를 1개 이상 제시

## Placeholder 규칙

- placeholder는 꼭 필요할 때만 쓴다
- `[목표 1개]`, `[파일 범위]`처럼 바꿔 넣을 위치가 명확해야 한다
- 미정 상태를 흐리는 모호한 표기는 금지한다

## 출력 형식 규칙

- 답변 형식을 요구할 때는 숫자 목록으로 고정한다
- 구현 프롬프트는 기본적으로 아래 형식을 권장한다
  1. 수정 파일
  2. 수정 내용
  3. 검증 결과
  4. 남은 리스크

## 다음 액션 링크 규칙

- 가급적 실제 파일명을 적는다
- 순서형 흐름이면 다음 번호 파일을 가리킨다
- 역할 전환이면 reviewer, browser verifier 같은 다음 역할을 명시한다
